## 🌐 Phase 4B: Federation Protocol

### Session 4: ActivityPub Foundation
**Files to create:**
- `lib/federation.ts` - Core federation logic
- `lib/db-federation.ts` - Federation tables
- `app/api/federation/inbox/route.ts` - Receive activities
- `app/api/federation/outbox/route.ts` - Send activities

**Database Schema:**
```sql
CREATE TABLE federation_instances (
  id TEXT PRIMARY KEY,
  domain TEXT UNIQUE NOT NULL,
  public_key TEXT NOT NULL,
  status TEXT DEFAULT "pending",
  created_at INTEGER DEFAULT (unixepoch())
);

CREATE TABLE federation_activities (
  id TEXT PRIMARY KEY,
  type TEXT NOT NULL,
  actor TEXT NOT NULL,
  object TEXT,
  target TEXT,
  instance_id TEXT,
  received_at INTEGER DEFAULT (unixepoch())
);
```

**ActivityPub Support:**
- Follow, Accept, Reject activities
- Create, Update, Delete for signals
- Actor profiles with WebFinger
- HTTP Signatures for auth

### Session 5: Cross-Instance Discovery
