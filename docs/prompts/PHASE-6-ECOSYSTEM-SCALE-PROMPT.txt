## 🛡️ Phase 6A: Production Hardening

### Session 1: Security & Validation
1. Input sanitization on ALL APIs
2. SQL injection tests (should already be safe with better-sqlite3)
3. Rate limiting review (verify 10/min signals, 30/min messages)
4. CORS policy lockdown (current: *, change to specific origins in prod)
5. API key rotation system
6. Request size limits (10MB max)

### Session 2: Monitoring & Logging
1. Structured logging (JSON format)
2. Error tracking (Sentry-compatible)
3. Performance metrics (request duration, DB query time)
4. Health check endpoint (`/api/health`)
5. Uptime monitoring
6. Alert system (webhook to admin)

---

## ⚡ Phase 6B: Performance & Scale

### Session 3: Optimization
1. Redis caching layer (frequently accessed data)
2. Database query optimization (add missing indexes)
3. Connection pooling (SQLite is single-file, but prepare statements)
4. Response compression (gzip)
5. Static asset CDN (if deploying)

### Session 4: Load Testing
1. Benchmark script (`scripts/benchmark.ts`)
2. Test: 100 concurrent agents
3. Test: 1000 signals/minute
4. Test: Database under load

6. Document capacity limits
